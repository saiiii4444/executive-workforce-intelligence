# Global Workforce Intelligence & Talent Optimization

## Overview
Global Workforce Intelligence & Talent Optimization transforms raw employee and recruitment data into actionable insights. The project helps HR leaders understand workforce dynamics, identify high‑potential talent, mitigate burnout risk and optimize recruitment and promotion. Python is used for data preparation and feature engineering; SQL for ad‑hoc queries; Power BI for interactive dashboards.

## Data Sources
This project blends several publicly available HR datasets (IBM HR Attrition, HRDataset_v14, aug_train/test and HR‑Employee‑Attrition) covering employee demographics, roles, performance scores, engagement surveys, absences, special projects count, salary and recruitment information. The raw CSVs are stored in `data_raw/` and cleaned outputs in `data_cleaned/`.

## Data Preparation & Feature Engineering
Data cleaning (duplicates removal, standardised dates, missing value imputation) and feature engineering are performed in `scripts/data_preprocessing.py`:
- **PerformanceScore Mapping:** Converts qualitative ratings (Exceeds, Fully Meets, Needs Improvement, PIP) to numeric scores.
- **Talent Score:** Composite metric (35% PerformanceScore, 25% EngagementSurvey, 20% EmpSatisfaction, 10% SpecialProjectsCount – 10% Absences).
- **Burnout Risk:** Risk proxy (30% Absences + 40% DaysLateLast30 – 30% EngagementSurvey).
- **Promotion Readiness:** Readiness metric (40% PerformanceScore + 30% SpecialProjectsCount + 30% EngagementSurvey).
- Recruitment dataset cleaning: fills missing company type/size, gender and education as “Unknown”, converts experience ranges to numeric and normalises training hours.

The script outputs three cleaned tables:
- `employee_talent_features.csv`
- `employee_attrition_clean.csv`
- `recruitment_clean.csv`

## Key Performance Indicators (KPIs)
Important metrics tracked in the dashboard:
- **Total Employees:** `COUNT(DISTINCT EmployeeNumber)`.
- **Average Salary:** `AVG(Salary)`.
- **Average Talent Score:** `AVG(Talent_Score)`.
- **Average Burnout Risk:** `AVG(Burnout_Risk)`.
- **Average Promotion Readiness:** `AVG(Promotion_Readiness)`.
- **Employee Satisfaction:** `AVG(EmpSatisfaction)`.
- **Attrition Rate:** `(Employees Who Left) / (Total Employees)`.
- **Recruitment Efficiency:** Combines offer acceptance, time to hire and candidate quality (custom formula).
- **Onboarding Completion Rate:** `Completed Onboarding / Total New Hires`.

## Dashboard Design
The Power BI dashboard is structured for executives:
1. **Top KPIs:** Cards and gauges summarise headcount, salary, talent health, burnout risk, promotion readiness and satisfaction.
2. **Department Overview:** Bar and column charts compare headcount, salary and satisfaction across departments.
3. **Talent vs Burnout:** Clustered column chart highlights departments with strong talent but high burnout.
4. **Promotion Readiness by Role:** Horizontal bar chart ranks roles by readiness.
5. **High Potential Employee Mapping:** Scatter plot visualises salary vs. talent score with bubble size representing promotion readiness.
6. **Employee Satisfaction Distribution:** 100% stacked bar shows satisfaction levels.

Interactive slicers (Department, Position and Gender) allow drill‑down analysis.

## How to Run
1. Clone the repo and set up a virtual environment.
2. Install dependencies (`pandas`, `numpy`, `scikit-learn`).
3. Run `python scripts/data_preprocessing.py` to clean and engineer data.
4. Import cleaned CSVs into Power BI to recreate the dashboard.
5. Run `scripts/department_summary.sql` on `employee_talent_features` for sample queries.
